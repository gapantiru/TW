1. Introducere

E necesara o aplicatie Web menita a gestiona eficient tranzactiile imobiliare. Sistemul va permite managementul unor imobile spre vanzare si/sau inchiriere, inclusiv informatii precum descriere, pret, coordonatele locatiei, date de contact, starea cladirii, facilitati oferite, riscuri posibile etc. Pentru localizarea facila, se va recurge la un serviciu de cartografiere (e.g., OpenStreetMap).

2. Architecture

Arhitectura de tip MVC.

3. Autentificare

Userul introduce email-ul si parola si daca sunt valide, se returneaza un JWT Token prin cookie, folosit pentru a accesa contul si anunturile.

4. Baza de date

MongoDB

5. Rest API

Permite filtrare dupa query string si implementeaza toate comenzile de inserare, put, post, update, delete.

Partea de adaugare anunt permite si adaugarea de imagini care vor fi trimise la server printr-un formData, care vor fi salvate apoi local intr-un folder cu id-ul utilizatorului.

Imaginile se sterg automat la stergerea anuntului respectiv.

Asigura si validarea unui user care incearca sa stearga un anunt, prin tokenul sau, validand faptul ca el este intr-adevar cel care a postat anuntul. Altfel, cererea este respinsa si se afiseaza un mesa j de eroare.

6. External API

Apelam serviciul Google pentru a face 3 mape: una statica (ia markere individuale plasate pe baza latitudinii si longitudinii), mapa globala, si mapa dinamica (poate modifica filedurile si pozitia markerului).
